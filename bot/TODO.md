# Ветка `bot` — статус и хендофф (Егор)

Telegram-бот на `aiogram` 3.x. Источник истины — `CONTRACT.md` (§4–§6), зона —
`CLAUDE.md`. Правлю только `/bot/` + `/tests/test_bot.py`, чужие модули зову по
публичному API (`db.*`, `parser.*`, `core.*`), их внутренности не трогаю.

## Что готово
- [x] `bot/config.py` — чтение `.env` (§8) + настраиваемые пороги/интервалы
- [x] `bot/mappers.py` — `DeadlineExtraction` → `Deadline` (чистая, тестируемая)
- [x] `bot/formatters.py` — форматирование ответов (день/неделя/дисциплина/зачёты)
- [x] `bot/handlers.py` — команды `/start /today /week /discipline /credits`,
      онбординг портала `/link` (FSM, пароль не логируется и удаляется), приём сообщений группы
- [x] `bot/scheduler.py` — APScheduler: синк расписания (готов) + напоминания (каркас)
- [x] `bot/main.py` — сборка Bot/Dispatcher, DI настроек, запуск планировщика + polling
- [x] `tests/test_bot.py` — маппинг + хендлеры на моках `db` (**pytest 6/6 зелёных**)
- [x] `bot/requirements.txt`, `.env.example`

## Как поднять локально (для всех в команде)
```bash
python -m venv .venv
# ВАЖНО: в этой среде pip ходит в сеть только в обход системного SOCKS-прокси:
NO_PROXY='*' .venv/Scripts/python.exe -m pip install -r bot/requirements.txt
.venv/Scripts/python.exe -m pytest tests/test_bot.py -v
```
Стек подтверждён рабочим: `aiogram 3.29`, `APScheduler 3.11`, `pydantic 2.13`, `pytest 9.1`.

### ⚠️ Про dev-стабы (читать обязательно)
Реальных `shared/core/db/parser` в ветке нет (см. блокеры), поэтому я положил
**временные заглушки в `_setup/stubs/`**. Они **не коммитятся** (`_setup/` в `.gitignore`),
а `tests/conftest.py` добавляет их в `sys.path` **в конец** — то есть как только в ветке
появятся настоящие модули, они автоматически получат приоритет, стабы отключатся.
Чтобы прогнать тесты у себя без моих стабов — нужны реальные модули ИЛИ свои заглушки.
`_setup/stubs/core/security.py` НЕ криптостоек (base64), это только для дева.

## ❗ Блокеры интеграции (нужна команда)
- **`shared/models.py` и `core/*`** (`deadline_extractor.py`, `llm.py`, `security.py`)
  **отсутствуют во всех ветках** (`main`, `bot`, `db`, `parser`), хотя контракт зовёт их
  `frozen`/«готов» (коммит «add AI core» их не содержит). Без них боевой ИИ и шифрование
  не работают. → найти реальный `core`/`shared` и влить в `main`.
- **`db/` (Настя)** есть только в `origin/db`. Для боевого запуска нужен поднятый Supabase
  и применённые миграции. ⚠️ В `origin/db` закоммичен реальный `.env` — убрать
  (`git rm --cached .env`) и **ротировать те секреты** (они скомпрометированы).
- **`parser/` (Максим)** ещё не реализован (`origin/parser` — только доки) → сквозной
  синк расписания заработает только после него. Тесты бота это не блокируют (моки).

## Секреты (`.env`, §8) — НЕ в репозитории
`.env` в `.gitignore`, каждый заполняет свой по `.env.example`. `FERNET_KEY` —
**один на проект** (должен совпадать у того, кто шифрует, и кто расшифровывает);
сгенерировать: `python -c "import os,base64;print(base64.urlsafe_b64encode(os.urandom(32)).decode())"`.
Для приёма сообщений группы в @BotFather: `/setprivacy` → **Disable**.

## ❓ Вопросы по контракту (дыры в API §6 — возможно PR `[contract]`)
1. **Напоминания без дублей.** В `db` нет «помечено как отправленное» — нужна
   функция/таблица состояния, иначе напоминания будут повторяться каждый скан.
2. **Дедлайны в личке.** `db.list_deadlines(chat_id, ...)` требует `chat_id` группы;
   связи user→группа нет. `/discipline`/`/credits` в личке не покажут дедлайны без неё.
3. **`/credits`.** Источник «зачёт/экзамен» — `kind` пары и `work_type` дедлайна?
   `db` не отдаёт `disciplines.control_form` напрямую. Подтвердить.
4. **Поля `DeadlineExtraction`.** Сверить точные имена (`discipline_name/work_type/
   due_at/raw_quote/confidence` + `is_actionable()`) по `core/deadline_extractor.py`.
