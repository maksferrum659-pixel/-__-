# CLAUDE.md — ветка `bot` (Егор)

## Перед началом
1. Прочитай `CONTRACT.md` — источник истины. Соблюдай §4 (поток данных), §5, §6.
2. Не трогай `shared/`, `core/`, `db/`. Работаешь **в `/bot/`**. Зависимости вызываешь по их публичному API.

## Твоя зона ответственности
Telegram-бот на `aiogram` 3.x: команды, приём сообщений чата, интеграция ИИ-ядра,
планировщик (синк расписания + напоминания). Ты — точка сборки: дёргаешь `parser`,
`core` и `db`, но их внутренности не редактируешь.

## Что реализовать
### `/bot/handlers.py` — команды (данные берём из `db`, не из портала напрямую):
- `/start` → `db.get_or_create_user(telegram_id)`; онбординг (попросить логин/пароль портала в личке).
- `/today`, `/week` → `db.get_schedule(telegram_id, since, until)` → форматированный ответ.
- `/discipline <name>` → расписание + дедлайны по дисциплине.
- `/credits` → зачёты/экзамены отдельным списком.
- Привязка портала: пользователь в **личке** присылает логин/пароль →
  `parser.authenticate` → `core.security.encrypt_token` → `db.save_portal_token`.
  Пароль нигде не логируем и не храним.

### `/bot/handlers.py` — приём сообщений группового чата (ИИ-поток):
- Бот добавлен в группу, **privacy mode выключен** (через BotFather: `/setprivacy` → Disable).
- На каждое сообщение группы: `core.extract_deadline(text, GigaChatClient(), date.today())`.
- Если `result.is_actionable()` и `confidence` приемлема → собрать `Deadline` (см. §5) → `db.upsert_deadline(...)` (дедуп по `chat_id`+`source_message_id`).
- Бот не видит историю до добавления — это ок, работаем «вперёд».

### `/bot/scheduler.py` — APScheduler:
- Периодический **синк расписания**: для каждого `db.list_users_with_token()` →
  `decrypt_token` → `parser.fetch_schedule` → `db.upsert_schedule_events`.
  Это и есть связка parser↔db — она живёт здесь, не в парсере.
- **Напоминания**: по `db.list_deadlines` и ближайшим парам — уведомления пользователю.

### `/bot/main.py`
Инициализация диспетчера, регистрация хендлеров, запуск планировщика, чтение `.env`.

## Тесты (`/tests/test_bot.py`)

- Маппинг `DeadlineExtraction` → `Deadline` корректен.
